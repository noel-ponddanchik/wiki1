First, determine whether any requestors have or would benefit from on-site consultations and make appointments as necessary.

Before leaving the office:

  • Prepare a list of sites to inspect. Use an existing list from previous/continuing field work, or create a report from the bike parking database: in Reports, 1 (new installs) or 2 (installed to verify). For #2, a work order can be used alternatively.
  • Map or plot out route if needed/desired
  • Take relevant list(s), clip board or binder, camera, measuring tape, wrench, nuts (for older verifications) cloth to wipe down rack tops, rack sticker(s), street map (if needed), reflective vest

In the field:

1. New installs
  • Survey location to determine whether a rack can be installed in compliance with guidelines
  • Spray dots to indicate location and make detailed notes on inspection sheet including: general location typically relative to the main business entrance; distance from curb and other landmarks, spacing between racks, if diagonal or perpendicular to curb (default is parallel—no need to note), and alignment relative to scores in the concrete (default is centered in square—no need to note)
  • If unsure, take a photo and discuss with Bicycle & Pedestrian Facilities Coordinator. (Unless you are the Bicycle & Pedestrian Facilities Coordinator.*)

2. Installed to verify
  • Grab rack and try to shake (to verify that it was securely installed)
  • Verify that rack was placed according to installer instructions
  • Verify that bolts do not extend more than ½” above nut
  • Tighten nuts if needed (or for older racks, install new nuts)
  • For grant funded racks, check that sticker(s) have been adhered to rack; if not, stick one on
  • Photograph each rack site using mobile phone.
  • If desired, photograph rack for another purpose (using file size appropriate to purpose)
  • Write date verified on work order form and any notes--including, for recently installed racks, "punch list" items that need to be corrected by the installer. If an older rack needs repair, also note.

Back in the office:

1. New installs
  • If a rack is recommended, add installer instructions, rack quantity, etc. in database. Installer instructions should be written in generally the same way/order (see other examples from database). The following default assumptions do not typically need to be recorded: inverted U, square tube, parallel to the face of curb (FC), and centered in sidewalk square. Instructions should include:
  1. # of racks (if not just one);
  2. type of rack (if not inverted U sq tube);
  3. general location with respect to business (if applicable);
  4. orientation of rack (if not parallel to FC);
  5. # inches from FC
  6. # inches between racks (if applicable)
  7. distance from obstructions (as applicable)
  • Abbreviations in the database to communicate rack location details include: ALFS (align left facing street); ARFS; straddles (meaning it straddles two squares); FC (face of curb)
  • If a rack is not recommended, change status to Not Installed, and explain why in the Notes field.
  • In both cases, document the Survey date and check the Inspected box.

2. Verified
  • Open \\oakland\pwa\Transportation\Bike-Ped_Program\Projects\Bike_Parking\database\BikeParking.accdb
  • Use qNeighborhood-Verification, make sure all fields are visible, search by neighborhood, rack ID#, or whatever is most helpful, and complete all fields.
  • In the photo field, use the following syntax (in this example 2125 is the rack ID#): 2125.jpg#database-rackphotos\2125.jpg. This creates a hyperlink from the databased to the photo. (Hint: click F2 in the photo field cell which will open up the source code window.)
  • If you have a question about information to be used in certain fields, please ask Jennifer.
  • If the install is incorrect or needs repair, advise Jennifer.
  • Label the photos by ID# and file in \\oakland\pwa\Transportation\Bike-Ped_Program\Projects\Bike_Parking\database\database-rackphotos.
  • Update the verified racks in the GIS layer: http://oakland-bike-ped.wikispaces.com/Update+Bike+Parking+and+Signage+GIS+Layers+After+Field+Work
  • The GIS bike parking layer will be updated twice a year, in January and July.
