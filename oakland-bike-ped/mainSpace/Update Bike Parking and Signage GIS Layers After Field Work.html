Purpose

When doing field verifications of installations or repairs of bike racks or wayfinding signs, one may come across instances where:
  1. the location of the bike rack or wayfinding sign does not match the location shown in the GIS layer, or
  2. there is no point in the GIS layer for the observed bike rack or wayfinding sign.
  3. Bi-annual update of layers

In these instances, one should update the layer.
Instructions
  1. Open associated layer .mxd in GIS (bike parking, signs, segments).
    1. For parking: \\oakland\pwa\Transportation\Bike-Ped_Program\GIS\parking\bike_parking_update_2017.mxd
  2. Make sure the layer is pulled from the Oracle server (GISADMIN.BikeFacilities) not the master.
  3. Make sure the Access database is up to date.
  4. Copy the "qBikeParking_SDE_Fields" query for parking or the "q1_Signs_SDE_Fields" query for signs into the csv (OaklandBikeParking_Current.csv) in \\oakland\pwa\Transportation\Bike-Ped_Program\GIS\data_share\Current-Geodatabase\Linked_Tables and save.
  5. Go back to ArcMap and right click the layer in the table of contents and join the layer to the updated csv based on RackID.
  6. Right click the layer, and click ‘Edit Features’ à ‘Start Editing’ to start an editing session for that layer.
  7. Check to make sure the rack or sign in question does not already have a point associated with it by opening the attribute table for the layer and searching for the correct ID of the rack or sign.
  8. Create needed points
    1. If there is no point for the rack or sign, go to EditoràEditing WindowsàCreate Features, and in the window which pops up, click the point symbol corresponding to the layer you want to edit.
    2. Move your cursor to where the point should be located and click to place the point.
    3. Click the "Attributes" button in the Editor tool bar and enter the correct value in the ID field for the rack or sign. If joined properly, the rest of the data should autofill.
    4. Repeat this process for the remaining racks or signs which need to have points created for them.
    5. Using the Editor menu, save your edits and click ‘stop editing.’
  9. Move existing points
    1. Find the point which needs to be moved
    2. Using the black arrow next to “Editor”, click and drag the point to its correct location
    3. Repeat this process for the remaining racks or signs which need to have their location changed.
    4. Using the Editor menu, save your edits and click ‘stop editing.'
  10. Check attribute table total count against count in access to ensure changes were made.
    1. may need to filter by status (installed vs removed)

For the Bi-Annual Update of Layers or as Needed

For Parking
  1. Run the "Update_Bike_Parking_From_Join_Model" to get the shapefile to autofill the attribute table from the joined table.
    1. Make sure you have the version of Bike Parking from the Prodgiso server not master.
    2. Start an editing session. IMPORTANT!: Otherwise you can't undo! Right click the layer, and click ‘Edit Features’ à ‘Start Editing’ to start an editing session for the Bike Parking layer
    3. Click the Catalog tab in the right of the page.
    4. Go to \\oakland\pwa\Transportation\Bike-Ped_Program\GIS\data_share\Current-Geodatabase then click the toolbox then double click on "Update_Bike_Parking_From_Join_Model" and click OK.
    5. The model should take about 45 minutes to run, but just let it do its thing and go about whatever else you were doing.
    6. Once finished, check to make sure the model worked by looking at the attribute table and ensuring fields that came up NULL before now pull from the join.
    7. Once satisfied, go to the Editor menu, save your edits, and click 'Stop Editing'.
