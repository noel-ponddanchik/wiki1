1. Batch Updates

This would be performed on occasions where many new photos are taken of newly installed racks/signs or of old racks/signs that needed to be verified. See section below the numbered instructions for details on how to update single entries in the photos field of a bike rack or wayfinding sign.

  1. Add your photos to the sign/rack database folder on the network.
  2. Within the folder of the sign/rack photos, press control + alt + shift + right-click and then select “Open command window here”
  3. At the command prompt, type “python create_picture_list.py” and then press enter. A message should print out saying that the tool finished creating the list of pictures. In the folder of photos, there should now be a pics.csv file.
  4. Open “pics.csv” in Microsoft Excel and in the column next to “forAccess”, paste in the SignID or RackID column from Microsoft Access.
  5. Next to the ID column created in step 4, create a new “Photos Field” column.
  6. Assuming “Photos Field” is in column E of Excel, that the ID column from Access is in column D of Excel, that "Match_Column" is column A of Excel, and that "forAccess" is in column C, type the following formula in the first empty row and then drag the formula down to apply it to all rows for which there is a RackID or SignID: =IFERROR(INDEX(C:C, MATCH(D2,A:A,0)), "")
  7. Copy and paste the resulting “Photos Field” column into the database.

2. Individual Updates

For making a new entry in the photo field without updating the entire column, simply type the rack number, “.jpg#database-rackphotos\”, the rack number again, and then “.jpg” into the appropriate cell. For example, type “xx.jpg#database-rackphotos\xx.jpg” where xx is replaced with the rack number whose photo field is being updated. For updating a particular sign photo cell in the sign photos database, use uppercase JPG and use “#SignPhotos\” instead of “#database-rackphotos\”.
