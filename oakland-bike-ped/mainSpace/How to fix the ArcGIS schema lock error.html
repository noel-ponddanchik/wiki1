The change in the network link for our data has created a recurring error in ArcGIS.

"failure to connect to database"
"cannot acquire a scheme lock due to an existing lock"

The fix is done by accessing the .mxd file in ArcCatalog to repair the links.

-first make sure no one is using ArcGIS on our team. If the geodatabase is being used you cannot make changes.

-navigate to the map
-right click the map
-click on "set data source"
-expand window
-change the beginning of all of the old links from \\pwafile2\transportation to \\oakland\pwa\Transportation
-a new map will be generated
-open the new map to ensure the fix worked
-then archive the old version in a "old" folder
-change the name of the new version from "copy" to the intended name
