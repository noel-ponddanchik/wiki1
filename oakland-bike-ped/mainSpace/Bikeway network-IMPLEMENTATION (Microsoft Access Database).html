This database was built for the 2007 update of the Bicycle Master Plan:
"\\oakland\pwa\Transportation\Bike-Ped_Program\BMP\2007_Plan-IMPLEMENTATION\Bikeways\Bikeway network-IMPLEMENTATION.accdb"

It was created in concert with the first build of the GIS layer that is still in use for the proposed bikeway network. The network is composed of 800+ segments, each with uniform characteristics along its length. Roadways are divided into multiple segments where those characteristics change. As a result, roadways with uniform cross-sections have segments that are long. In comparison, some roadways change cross-section each block and thus have short segments. As field conditions change, it may be necessary to divide one segment into multiple segments in order to capture the new changes in condition. Those segments are created in the database and then ID numbers from the database are used for revised line segments in the GIS layer.

Objects


Tables

  • tBikeway_Segments: This is the main table of data from which almost everything else flows. See tSegmentLengths below for the other table of source data.
  • tLookup[…]: The 12 tables beginning with “tLookup” create predefined values and drop-down menus for key fields in the tBikeway_Segments table.
  • tSegmentLengths: These are lengths (in feet) of the roadway and path segments in tBikeway_Segments. The lengths are exported from GIS and matched by ID number. This table needs to be updated as segment extents are modified or added in tBikeway_Segments and the GIS layer. Additional information on this synchronization (and associated troubleshooting) is in the documentation for the timelapse map and database.

Queries

  • q-Bikeway_Segments-mileage: Provides the length of segments by linking "tBikeway_Segments" to "tSegmentLengths”. It is used by the “rBikeway_Segments” report.
  • qBikewaySummary (and the related queries “_Divided”, “_Existing”, “_OneWay”, _OneWayOnly”, “_Proposed”, “_TwoWay”): Provides the necessary link between "tBikeway_Segments" and "rBikeway_network_mileage_summary" to produce mileage totals for existing/proposed bikeways by bikeway class.
  • qProject_mileage1: Associates segment mileage with the projects (composed of multiple segments) that were used in the project prioritization for the 2007 Bicycle Master Plan.
  • qProject_mileage2: Sums the mileage of individual project segments from "qProject_mileage1" into total project lengths used in the project prioritization for the 2007 Bicycle Master Plan.
  • qTmp: A temporary object for doing one-off queries.

Reports

  • rBikeway_network_mileage_summary: Generates existing and proposed bikeway mileage by class in two tables: one by roadway centerline miles and one by roadway directional miles. The report depends upon the following queries:
    • qBikewaySummary
    • qBikewaySummary_Divided
    • qBikewaySummary_Existing
    • qBikewaySummary_OneWay
    • qBikewaySummary_OneWayOnly
    • qBikewaySummary_Proposed
    • qBikewaySummary_TwoWay
  • rBikeway_Segments: Generates a summary of select fields in tBikeway_Segments for all records. The report depends upon the following query to generate mileage:
    • q-Bikeway_Segments-mileage
