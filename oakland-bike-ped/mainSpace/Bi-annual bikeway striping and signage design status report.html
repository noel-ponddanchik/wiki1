The bi-annual striping and signage update includes a map and pages from the striping and signage projects tracking spreadsheets in
\\oakland\pwa\Transportation\Bike-Ped_Program\Projects\Bikeways-project_tracking. The packet is typically created in time for the April and October BPAC meetings, and is also posted on the web, replacing asset oak026930; http://www2.oaklandnet.com/Government/o/PWA/o/EC/s/BicycleandPedestrianProgram/OAK024586 needs to be updated to reflect the current date. Jennifer compiles the packet.

Update .mxd

\\oakland\pwa\Transportation\Bike-Ped_Program\Projects\!Tracking\design-status-map\Bikeways_CurrentStripingSignage.mxd

1. Ask Jennifer/Jason to confirm that that Access Database table "qBikewaysCurrentGIS" query of
\\oakland\pwa\Transportation\Bike-Ped_Program\GIS\bikeways\Segments-database\BikewaySegments.accdb
has been updated with the most recent bikeway information. Copy and paste the entire query into
\\oakland\pwa\Transportation\Bike-Ped_Program\GIS\data_share\Current-Geodatabase\Linked_Tables\OaklandBikewayNetwork_current.xls
    1. In the new .xls, run the Macro. This Macro replaces the numbers listed for Bikeway Class (1, 2, 3, etc.) with descriptive names (bike path, bike lane, bike route).
      1. To run the macro you should enable the developer tab by going to File > Options > Customize Ribbon and check the Developer box then press OK
      2. Then go to the Developer tab and click Macros the click on the Macro "NetwrkMacro.xls!NetworkConversion and click Run.
      3. Save the resulting updated bikeway network

2. Open the .mxd file which should include:
  • Striping Status (from the Bikeways layer, symbolized by Striping Status)
  • Signage Status (from the Bikeways layer, symbolized by Signage Status)
    • Make sure the Signage Status 'Installed' does not include streets with numbered bike route signs (SG45s) by symbolizing by Multiple Attributes, making the first Value Field 'SignageStatus' and the second 'SignType' and group by installed, designed, and in design, exclude 'installed, SG45' from the installed group.
  • Bikeways (symbolized by existing class, excluding 0 in black and proposed class, excluding 0 in grey)
  • Priority Project Status (symbolized as yes = 'yogo blue' size 8pt, 50% transparency)
    • Do not include projects that have already been striped by going to the Priority Project Status Layer Properties, go t Definition Query, and type: "Sheet1$.StripingStatus" = '' OR "Sheet1$.StripingStatus" = 'designed' OR "Sheet1$.StripingStatus" = 'in design'
  • Adjacent Bikeways

(If any of these are not already in the mxd, you can add the Bikeway Network layer from the Oracle Server and symbolize as it's described below.)

See details on the symbology and other map layout elements below.

3. Confirm that the "Striping Status", "Sign Projects", "Other Existing Bikeways" and "Other Proposed Bikeways" layers are joined to "Sheet1$" (the sheet in OaklandBikewayNetwork_current.csv). You should not have to rejoin the database table to the layer, as once there is a join relationship, the layer will update automatically. If it is not already, join to the table you just updated at:
\\oakland\pwa\Transportation\Bike-Ped_Program\GIS\data_share\Current-Geodatabase\Linked_Tables\OaklandBikewayNetwork_current.csv


4. Change the date in the map title.

5. Set bookmark to citywide extent. Export the map to a PDF and save in \\oakland\pwa\Transportation\Bike-Ped_Program\Projects\Bikeways-project_tracking\design-status-map\Completed Projects Status Maps, Name file with date of map (example: "Bikeways_CurrentStripingSignage-10-2013.pdf")

6. Spot check for errors, particularly missing or mis-labeled segments. If everything looks good, print an 11 x 17 map using the color printer.



Symbology and other map element details

Striping Status Layer
Symbolize using the Bikeways Layer, copied and displaying only 'Striping Status' from the attribute table.
  • Installed (Width 2, Peacock Green) /Advanced Symbol Level 1
  • Designed (Width 2, Medium Olivenite) /Advanced Symbol Level 2
  • In Design (Width 2, RGB Color: R 255, G 250, B 125) / Advanced Symbol Level 2

Signage Status Layer
Symbolize using the Bikeways Layer, copied and displaying only 'Signage Status' from the attribute table.
  • Installed (Width 1.25 x2 lines: First line offset 1.5/ Second line offset -1.5)
  • Designed (Width 1.25 x2 lines: First line offset 1.5/ Second line offset -1.5 /Template (dashed line):
  • 7 shaded black, gray square 10 unites away, Interval 1.00)
  • In Design (Width .80 x2 lines: First line offset 1.5/ Second line offset -1.5 /Template (dashed line):
  • 5 shaded black, gray square 9 units away, Interval 1.00)

Bikeways Layer
Proposed Bikeways: Change layer name to "Other Proposed Bikeways."
  • Symbolize by "ProposedClass" (NOT "PROPOSEDCL"), excluding 0 and 4. (Width 1.0, color 40% Gray)
Existing Bikeways: Change layer name to "Other Existing Bikeways."

  • Symbolize by "ExistingClass" (NOT "EXISTINGCL"), excluding 0 and 4. (Width 1.0, color Black)

Priority Project Layer
Symbolize by "PriorityProject" - "yes" ('yogo blue' Width 8.0)
Right click on the "Priority Project Status" layer then click properties then click the Display tab and make the transparency 50%.
