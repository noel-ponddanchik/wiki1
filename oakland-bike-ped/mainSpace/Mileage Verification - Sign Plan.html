Mileage Verification

  • Measure distances, to the nearest tenth of a mile, from destinations to the nearest confirmation sign, and subsequently between each following confirmation sign
  • Look at existing neighboring signs to confirm network continuity.
  • Distances can be measure using Google's pedometer tool at http://www.gmap-pedometer.com/
  • Distances can also be measured using ArcMap's measure tool. If the measure tool is not showing in the toolbar, go the Customize>Toolbars and make sure that 'Tools' is checked. To change the distance units to miles click the capital sigma button within the Measure window.
  • Distances should be consistent along the network. For example, if Sign A lists the following distances:
    Oakland Tech HS 0.5
    California College of the Arts 0.8
    [ba] Rockridge 1.3
    Sign B, 0.3 mi further along the bikeway, should have these distances:
    Oakland Tech HS 0.2
    California College of the Arts 0.5
    [ba] Rockridge 1.0
  • Note that the distances between destinations in the above example are consistent on both assemblies by measuring between signs and using that distance to get the mileage for each destination on the sign (if it's .3 miles between signs, each destination's mileage should be .3 miles different from the sign before it, as in the above example).
  • Update the applicable records in the database with the proper mileages (replacing the “x.x”s). It’s particularly useful to have your mileage work checked by someone else. Update the sign labels in the plan to agree with the mileages.
