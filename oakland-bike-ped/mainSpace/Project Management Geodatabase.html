Selecting Bikeway Segments by Project Polygon for Editing Field Values

Workflow Overview:
Isolate the project polygon> Use this polygon to select the project's bikeway segments> Apply edits
  • First, double click on bikeprojectmanagement layer in the table of contents. In the Layer Properties, navigate to the Definition Query tab.
    • Click the Query Builder button, double click ProjectID, press the = button, and the Get Unique Values button.
    • Double click on the project value you are interested in selecting the segments within.
    • Your 'SELECT*FROM bikeprojectmanagement WHERE:' query should be 'ProjectID = 5" (5 being an example project polygon of interest).
    • Press OK, you should now only see the project polygon of interest displayed in the map.
  • Now go to Select at the top menu bar and choose Select By Location. Then enter the following:
    • Selection method: select features from
    • Target layer(s): bikeways
    • Source layer: bikeprojectmanagement
    • Spatial selection method for target layer feature(s): are within the source layer feature
    • Press Apply and you should see the bikeway segments within the project polygon highlighted
    • Click OK to exit this screen
  • With the segments selected you can now edit field values withing the bikeways table.
    • Right click the bikeways layer in the table of content and click Open Attribute Table
    • Click Show Selected Records button to see only the highlighted segments in the table
    • To begin editing values in the table, go to the Editor toolbar and click Start Editing. Choose the bikeways layer.
    • Now you can double click any field value and change the value. Or use the Field Calculator to do batch edits
    • When you are done editing, go to the Editor toolbar and choose Stop Editing. Make sure to save your edits

This layer contains all bikeways joined to pci_2013. Missing road sections from pci_2013 were copied from GISADMIN.OaklandStreets.
<span style="font-size: 14pt;">Data Dictionary of Bike/Ped Program Fields</span>
<span style="font-weight: bold;">BIKEID_1 </span>: Bikeway segment ID assigned by Bike Ped Program
<span style="font-weight: bold;">PrimaryBike</span>: (Y/N) Y if bikeway segment is a primary route
<span style="font-weight: bold;">ProposedClass</span>: (0, 1, 2, 2.1, 3, 3A, 3B, 4) The proposed bikeway class*
* No bikeway (Class 0)
* Bike Path (Class 1)
* Bike Lane (Class 2)
* Bike Route (Class 3)
* Arterial Bike Route (Class 3A)
* Bicycle Boulevard (Class 3B)
* Proposed bikeway designated for improvements by BMP (Class 4)
<span style="font-weight: bold;">ExistingClass </span>: (0, 1, 2, 2.1, 3, 3A, 3B, 4) The existing bikeway class*
* No bikeway (Class 0)
* Bike Path (Class 1)
* Bike Lane (Class 2)
* Bike Route (Class 3)
* Arterial Bike Route (Class 3A)
* Bicycle Boulevard (Class 3B)
* Proposed bikeway designated for improvements by BMP (Class 4)
<span style="font-weight: bold;">StripingStatus</span>: (designed, in design, installed) Design and installation status of bikeway
 
 
<span style="font-weight: bold;">*</span><span style="font-style: italic;">Extra Bikeway Class Detail</span>
Value: 0 = No bikeway
* Description: Either no bikeway exists or no bikeway is proposed for a particular roadway segment. For Existing Class 0, the Proposed Class is generally not 0. In some instances an existing bikeway (Existing Class does not equal 0) is proposed to be removed (Proposed Class 0). Segments with both Existing Class 0 and Proposed Class 0 were analyzed as part of the 2007 Bicycle Master Plan but not included as part of the bike network.
Value: 1 = bike path
* Description: Provides a paved right-of-way that is completely separated from the street. Bike paths are typically shared with pedestrians and are often called mixes-use paths.
Value: 2= bike lane
* Description: Striped on streets and designated specifically for bicyclists. They are Oakland's preferred treatment for busy streets.
Value: 2.1 = Bike lane on one side only
* Description: This designation is used for one bike lane on only one side of a two-way street. Note that a one-way street with one bike lane is designated as Class 2. Class 2.1 is used to imply unfinished business - a bike lane is still needed on the street.
Value: 3 = bike route
* Description: Have shared travel lanes and are designated because they provide better connectivity than other nearby streets. Bike routes are designated with bikeway guide signs and may have no other bicycle-specific features. The 2007 Bicycle Master Plan proposes bike routes only in the Oakland Hills above Mountain Blvd due to the recreational nature and challenging topography of this area. The bikeways in the rest of the city are proposed for bikeways that provide a higher level of accommodation (Class 1, 2, 3B).
Value: 3A = arterial bike route
* Description: Located on busier streets where bike lanes are not possible and parallel streets do not provide adequate connectivity. They include sharrows and bicycle-specific signage. Arterial bike routes are used to make near-term connections in the bike network. Over time they should be replaced with better facilities (e.g., Class 2).
Value: 3B = bicycle boulevard
* Description: Prioritizes bicyclists on quiet residential streets with traffic calming, sharrows, and bicycle guide signs.
Value: 4 = Proposed bikeway
* Description: This class is an artifact of the "I [bike] Oakland" map to identify proposed bikeways with wide travel lanes or low traffic volumes that provide useful connections in their current form. These streets are generally proposed for Class 2 or Class 3B facilities. These Class 4 bikeways are tracked in the GIS layer as a subset of all proposed bikeways so as mileage can be computed for the cover of the "I [bike] Oakland" map.
