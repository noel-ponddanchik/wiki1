Pre-defined queries in Access facilitate data entry and work flow. They are not required--all work can be done from the main table--but they show the key information for the work phase, providing clarify in an otherwise cluttered database. Queries include a subset of all the fields in the main table (tBikeParking)--those relevant to the particular phase in the work flow, and other identifying fields. It is important to complete every relevant field in each query--otherwise, subsequent queries and/or reports will be incomplete. (Some fields are not "relevant" but remain as identifiers.)

Key queries and reports (as of February 2016):
  • q1 Enter Request: (1) enter new requests (feeds into "r1Locations to Inspect" report for field inspections) AND (2) change Timeline status to Inspected after inspections
  • q2 Field Inspection (enter installation instructions, number of racks, racks type(s), survey date and other relevant info after location has been surveyed)
  • q3Mailing (prepare envelopes for letters to business owners; also change the Status to Mailed and enter Respond by date)
  • q3ToVerify (once racks have been installed; FIRST change the Status from In Progress to Installed)
  • q4 Work Order. This query is used at two points: (1) to generate work orders for installations, and (2) to change the status from In-Progress to Installed and teh Timeline to "Installed to verify" after racks are installed. Records only appear in this query if the site has been inspected AND the response deadline has passed. These milestones mean that the location is ready to be included in a new work order. To make this work, change the Timeline status to Work Order, and check the Work Order check box. [In the future it would be good to get ride of the check box--for now, queries and reports depend on both. I think.]
  • q3ToVerify: a report with records that have been installed. After these locations have been verified in the field, use the field verification query (Q3) and fill in all the fields.
  • qNeighborhood-Verification (document installation and ensure all needed information is entered into the database. Make sure every field is completed (except where there is no relevant information, like the Notes or Suite Number fields).

For ongoing verifications, see \\oakland\pwa\Transportation\Bike-Ped_Program\Projects\Bike_Parking\1-HowTo\installs-and-verification.doc. For neighborhood verification projects, see \\oakland\pwa\Transportation\Bike-Ped_Program\Projects\Bike_Parking\1-HowTo\rack-neighborhood-verification.doc.

Defunct Reports:
r3 Work Order-New Installs: creates a work order for any sites that have been field inspected AND the letter response deadline of two weeks has passed.
r4 Work Order - Repairs: generates a list of sites that have been determined to be in need of repairs. The Repair checkbox should be checked and the status changed to Needs Repair. They will remain in this report until the status has been restored from Needs Repair to Installed and the Repair checkbox has been unchecked.
