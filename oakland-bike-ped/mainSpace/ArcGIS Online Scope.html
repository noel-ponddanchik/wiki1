Goals

  • To create a browser environment for editing our geodatabases with tools available from ArcGIS online
  • Create mobile app for field work
  • Web maps for public

Opportunities for Process Improvements

  • Serve data from a single source, eliminating where we have multiple sets of books that need to be updated manually to keep them in sync with each other.
  • Manage bikeway striping as an asset whereby individual bikeway segments are associated with each other to become projects; projects are developed and implemented; and the results of those projects are captured as assets. Currently, this is a situation with two sets of books: one for bikeway segments (in Access) and another for bikeway projects (in Excel).
  • Create the ability to edit data through a map interface without the need for fluency in ArcMap.
  • Manage the bikeway network at a finer level of detail (e.g., by block and by direction, rather than longer segments as is currently done).
  • Make available to city staff key data that we own and maintain; make that data easy to update and to share.
